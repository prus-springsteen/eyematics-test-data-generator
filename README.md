# EyeMatics Test Data Generator

A simple tool to generate (fake) test data for the EyeMatics project. The tool is written in Java 17 and uses the HAPI FHIR 
library (Version 5.1.0). 

## About

This software project is part of the [EyeMatics](https://www.eyematics.org/) project. It is developed for the use of
test data exchange using the [EyeMatics Data Sharing Framework Plugin](https://github.com/prus-springsteen/eyematics-dsf-process-plugin).
If you are interested in the DSF, please visit the [DSF website](https://dsf.dev). This project contains a simple command 
line application that generates test data for the EyeMatics project, based on Builder classes for creating EyeMatics 
FHIR resources.

The Observation, DiagnosticReport, Medication, MedicationAdministration and MedicationRequest resources are generated according to the 
[Eyematics Core Data Set](https://imi-ms.github.io/eyematics-kds/).  Furthermore, the MeasureReport resources are generated with an EyeMatics associated profile (https://eyematics.org/fhir/MeasureReport/ivi-patients-summary).

The Consent resource is generated according to the [MII IG Consent v2025](https://simplifier.net/guide/mii-ig-modul-consent-2025?version=2025.0.4) 
standard.

## Prerequisites

1. Install [git](https://git-scm.com/book/en/v2/Getting-Started-Installing-Git)
2. Clone this [GitHub repository](https://github.com/prus-springsteen/eyematics-test-data-generator)
3. Change directory to the cloned GitHub repository (working tree)

## Build the Application

This project uses **Maven** as its build tool. Test data can either be generated by building the project and running the 
resulting JAR file or by running the applications "main" directly from the IDE.

### Build the JAR

Run the following command in the project root:

``` bash
mvn clean package
```

After the build completes, the `target/` directory will contain:

``` text
eyematics-test-data-generator-<version>-SNAPSHOT.jar
```

## Running the Application

``` bash
java -jar target/eyematics-test-data-generator-<version>-SNAPSHOT.jar
```

The application will generate test data and print the steps to the console. The generated data can be found in the 
`output/` directory.

## Requirements

-   Java 17+
-   Maven 3.6+
